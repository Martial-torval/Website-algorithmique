Tests conditionnels

Si Sinon

Dans le test Si FinSI, les conditions renvoient obligatoirement à un booléen ou une expression booléenne. Les tests conditionnels peuvent s’emboiter grâce à l’expression SinonSi. Dans ce cas l’indentation est « en cascade ».

Selon Cas

Selon Cas teste des conditions sur des variables de type entier. deux autres syntaxe sont à connaître :
QuitterSelon est optionnel, si les cas est vérifié, il permet de sortir du test conditionnel.
parDefaut renvoie à une instruction qui est exécutée si la condition n'est pas vérifiée dans aucun des cas. Il est optionnel.

Tant Que

TantQue teste des conditions sur des variables de type booléen sous forme de boucle, tant que la condition n'est pas remplie, la boucle va continuer de tourner jusqu'a ce que la condition sois remplie. Tout ceci se compose en trois parties : condition de continuation : Nous allons définir la condition sur laquelle la boucle va tournée ou non. instructions : Nous allons définir les instructions à effectuer pour chaques tour de boucles. incrémentation : Nous allons ajouter 1 à l'indice i pour refaire un tour de boucle tant que l'indice i ne dépasse le booléen inscrit dans la condition de continuation.

Nous voulons afficher 100 fois le même message
Début
tantQue i < 101
afficher("j'écris ce message 100 fois")
i = i + 1
finTantQue
fin
